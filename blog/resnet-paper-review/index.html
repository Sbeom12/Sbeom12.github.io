[논문 리뷰] Deep Residual Learning for Image Recognition (ResNet)

ILSVRC 2015 우승 모델 ResNet의 핵심 아이디어인 Residual Learning을 분석하고, Bottleneck, ResNeXt까지 확장하여 정리합니다. PyTorch 구현 코드 포함.

AI Vision 논문리뷰 ResNet Deep Learning

Series: Vision Model

  1. 1. [논문 리뷰] Deep Residual Learning for Image Recognition (ResNet)

개요

Kaiming He, Xiangyu Zhang, Shaoqing Ren, Jian Sun (Microsoft Research, 2015) 논문 링크

ResNet은 ILSVRC 2015 우승 모델로, VGG보다 8배 깊은 152층 네트워크임에도 3.57%라는 놀라운 오류율을 달성했습니다. 핵심 아이디어는 단순하지만 강력합니다: Residual Learning.

Introduction: 깊어질수록 좋아지는가?

이전 연구들(“Going deeper with convolutions”, “VGGNet”)은 네트워크가 깊을수록 성능이 좋아진다고 가정했습니다. 하지만 실제로는 Gradient Vanishing/Exploding 문제가 발생합니다.

  • Vanishing: 역전파 시 gradient가 점점 작아져 앞쪽 레이어가 학습되지 않음
  • Exploding: 가중치의 반복 곱셈으로 gradient가 발산

위 그래프에서 56층 네트워크가 20층보다 학습 오류와 테스트 오류 모두 높은 것을 확인할 수 있습니다. 이는 오버피팅이 아닌 최적화 문제입니다.

Residual Learning Framework

핵심 아이디어

기존 네트워크가 목표 함수 H(x)를 직접 학습하는 대신, 잔차(residual) F(x) = H(x) - x를 학습합니다.

$$H(x) = F(x) + x$$

왜 이것이 효과적인가?

  • Pre-conditioning: 최적 상태에서 F(x) ≈ 0이 되면 항등 함수에 수렴
  • 민감도 향상: 입력의 미세한 변화를 감지 가능
  • 최적화 용이: 깊은 네트워크도 학습 가능
  • 성능 보장: 레이어를 추가해도 성능이 하락하지 않음

Identity Mapping by Shortcuts

Identity Shortcut (차원이 같을 때)

$$y = F(x, {W_i}) + x$$

  • 첫 번째 항: 학습되는 residual mapping
  • F + x: element-wise 덧셈 (shortcut connection)
  • 단순화하면: $F = W_2 \sigma(W_1 x)$ (σ는 ReLU)

Projection Shortcut (차원이 다를 때)

$$y = F(x, {W_i}) + W_s x$$

  • Ws: 1×1 Convolution으로 차원을 맞춰주는 행렬
  • 역전파 시에도 Ws 파라미터를 통해 gradient가 전달되므로 vanishing 방지

아키텍처 비교

VGG19

  • 13 Convolution + 3 Fully-connected 레이어
  • 3×3 conv 필터, stride 1, padding 1
  • 2×2 max pooling (stride 2)
  • ReLU 활성화 함수

Plain Network

VGG를 기반으로 설계 규칙을 적용:

  1. 같은 크기의 feature map에는 같은 수의 필터 사용
  2. Feature map 크기가 절반이 되면 필터 수를 2배로
  • Pooling 대신 stride 2 convolution으로 다운샘플링
  • 마지막에 Global Average Pooling 적용
  • 34층이지만 VGG19보다 연산량이 적음

Residual Network

Plain Network에 shortcut connection을 추가한 구조.

차원 맞춤 전략 3가지:

  1. Zero-padding: 추가 파라미터 없이 padding으로 차원 맞춤
  2. 선택적 1×1 Conv: 크기가 변하는 shortcut에만 적용
  3. 전체 1×1 Conv: 모든 shortcut에 convolution 적용

실험 결과

핵심 발견:

  • Plain-18이 Plain-34보다 성능이 좋음 → degradation 발생
  • ResNet-34가 ResNet-18보다 성능이 좋음 → Residual Learning으로 해결
  • Residual Network의 수렴 속도가 Plain Network보다 빠름

Bottleneck Architecture

더 깊은 네트워크(50/101/152층)를 위해 차원 축소-확장 구조를 도입:

  • 1×1 conv로 차원 축소 → 3×3 conv → 1×1 conv로 차원 확장
  • 연산 시간 단축 + 파라미터 22% 감소

파라미터 비교:

  • Standard: 3×3×64×2 = 1,152
  • Bottleneck: 1×1×64 + 3×3×64 + 1×1×256 = 896

최종적으로 6개 모델 앙상블로 3.57% 오류율을 달성했습니다.

CIFAR-10 실험

32×32 이미지에 맞게 첫 번째 7×7 conv를 3×3으로 조정하여 실험을 진행했습니다.

  • Plain Network는 일정 깊이 이상에서 성능이 정체
  • ResNet은 깊이가 증가할수록 성능 향상 (실용적 한계 내에서)
  • 1202층 결과는 regularization 없이 오버피팅 발생

Pre-activation (2016 후속 연구)

BN-ReLU-Conv 순서로 변경하여:

  • 일반화 성능 향상
  • 최적화 단순화
  • 1000층 이상에서도 효과적

추가 연구: ResNet의 앙상블 행동

Residual Networks Behave Like Ensembles of Relatively Shallow Networks

Lesion Study

핵심 발견:

  • VGG는 레이어 하나를 제거하면 성능이 급격히 하락
  • ResNet은 다운샘플링 지점을 제외하면 안정적으로 유지
  • ResNet은 다수의 독립적인 경로(앙상블)로 동작

경로 분석

  • 95%+ 경로가 19~35개 모듈을 통과
  • Gradient 업데이트는 주로 5~17 길이의 경로(~45%)에서 발생
  • 20개 이상의 모듈을 거치는 경로는 효과적인 학습이 어려움

결론: ResNet은 “깊은 네트워크”가 아니라 “얕은 네트워크의 앙상블”로 동작합니다.

Bottleneck 효과 분석: Inception과 Xception

1×1 Convolution의 역할

Google Inception에서 도입된 1×1 conv의 장점:

  • 채널 수 조정
  • 연산량 감소
  • 비선형성 추가

단점: 채널 축소 시 정보 손실 발생

Inception vs Xception

발견: 1×1 Convolution을 많이 쓴다고 반드시 성능이 좋아지는 것은 아닙니다.

참고: Xception: Deep Learning with Depthwise Separable Convolutions

ResNeXt: Aggregated Residual Transformations

논문 링크

ResNeXt는 ResNet에 Cardinality(그룹 수) 개념을 도입합니다:

  1. 256 채널 → 1×1 conv → 128 채널
  2. 32개 그룹으로 분할 (각 4 채널)
  3. 32개 그룹 출력 concat → 128 채널
  4. 1×1 conv → 256 채널

Cardinality vs Width

32그룹 × 4채널 > 더 넓은 설정

스케일링 전략 비교

순위: ResNeXt-101 > ResNet-101 (wider) > ResNet-200

핵심 결론: Cardinality 스케일링 > Width 스케일링 > Depth 스케일링

PyTorch 구현

Residual Block

import torch
import torch.nn as nn

class Residual_Block(nn.Module):
    def __init__(self, input, output, stride=1):
        super(Residual_Block, self).__init__()

        self.conv1 = nn.Conv2d(input, output, kernel_size=3,
                              stride=stride, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(output)
        self.relu = nn.ReLU(inplace=True)

        self.conv2 = nn.Conv2d(output, output, kernel_size=3,
                              stride=1, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(output)

        self.shortcut = nn.Sequential()
        if stride != 1:
            self.shortcut = nn.Sequential(
                nn.Conv2d(input, output, kernel_size=1,
                         stride=stride, bias=False),
                nn.BatchNorm2d(output)
            )

    def forward(self, x):
        out = self.bn1(self.conv1(x))
        out = self.relu(out)
        out = self.bn2(self.conv2(out))
        out += self.shortcut(x)  # skip connection
        out = self.relu(out)
        return out

ResNet 모델

class ResNet(nn.Module):
    def __init__(self, block, num_blocks, num_classes=10):
        super(ResNet, self).__init__()
        self.in_planes = 64

        self.conv1 = nn.Conv2d(3, self.in_planes, kernel_size=3,
                              stride=2, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(64)
        self.relu = nn.ReLU(inplace=True)
        self.maxpool = nn.MaxPool2d(kernel_size=3, stride=2, padding=1)

        self.layer1 = self._make_layer(block, 64, num_blocks[0], stride=1)
        self.layer2 = self._make_layer(block, 128, num_blocks[1], stride=2)
        self.layer3 = self._make_layer(block, 256, num_blocks[2], stride=2)
        self.layer4 = self._make_layer(block, 512, num_blocks[3], stride=2)
        self.linear = nn.Linear(512, num_classes)
        self.avgpool = nn.AdaptiveAvgPool2d((1, 1))

    def _make_layer(self, block, planes, num_blocks, stride):
        strides = [stride] + [1] * (num_blocks - 1)
        layers = []
        for stride in strides:
            layers.append(block(self.in_planes, planes, stride))
            self.in_planes = planes
        return nn.Sequential(*layers)

    def forward(self, x):
        out = self.bn1(self.conv1(x))
        out = self.relu(out)
        out = self.maxpool(out)

        out = self.layer1(out)
        out = self.layer2(out)
        out = self.layer3(out)
        out = self.layer4(out)

        out = self.avgpool(out)
        out = out.view(out.size(0), -1)
        out = self.linear(out)
        return out

def ResNet18():
    return ResNet(Residual_Block, [2, 2, 2, 2])

def ResNet34():
    return ResNet(Residual_Block, [3, 4, 6, 3])

학습 및 평가

import torchvision
import torchvision.transforms as transforms
import torch.optim as optim
import torch.backends.cudnn as cudnn

# 데이터 로드
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
])

train_dataset = torchvision.datasets.CIFAR10(
    root='./data', train=True, download=True, transform=transform)
test_dataset = torchvision.datasets.CIFAR10(
    root='./data', train=False, download=True, transform=transform)

train_loader = torch.utils.data.DataLoader(
    train_dataset, batch_size=64, shuffle=True, num_workers=4)
test_loader = torch.utils.data.DataLoader(
    test_dataset, batch_size=50, shuffle=False, num_workers=4)

# 모델 설정
device = 'cuda'
model = ResNet18().to(device)
model = torch.nn.DataParallel(model)
cudnn.benchmark = True

criterion = nn.CrossEntropyLoss()
optimizer = optim.AdamW(model.parameters())

# 학습 루프
for epoch in range(20):
    model.train()
    for inputs, targets in train_loader:
        inputs, targets = inputs.to(device), targets.to(device)
        optimizer.zero_grad()
        loss = criterion(model(inputs), targets)
        loss.backward()
        optimizer.step()
