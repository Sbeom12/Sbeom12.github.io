[BOJ 1002] 터렛 — 두 원의 교점, 엔지니어링적 사고로 풀기

백준 1002번 터렛 문제를 단순 구현이 아닌, 가진 자원 분석 → 문제 모델링 → 케이스 분류 → 검증까지 엔지니어링 관점에서 풀어봅니다.

알고리즘 기하학 Python BOJ

문제 요약

백준 1002번 - 터렛

두 터렛의 좌표 (x1,y1)(x_1, y_1)(x1​,y1​), (x2,y2)(x_2, y_2)(x2​,y2​)와 각 터렛에서 적까지의 거리 r1r_1r1​, r2r_2r2​가 주어질 때, 적이 있을 수 있는 좌표의 수를 구하는 문제.

Step 1: 내가 가진 자원은 무엇인가

코드를 작성하기 전에, 이 문제에서 내가 가진 것과 구해야 하는 것을 정리합니다.

주어진 것 (Input):

  • 두 점의 좌표: (x1,y1)(x_1, y_1)(x1​,y1​), (x2,y2)(x_2, y_2)(x2​,y2​)
  • 각 점에서 목표까지의 거리: r1r_1r1​, r2r_2r2​

구해야 하는 것 (Output):

  • 목표가 있을 수 있는 위치의 수 (0, 1, 2, 또는 −1-1−1 = 무한대)

내가 활용할 수 있는 도구:

  • 두 점 사이의 거리 공식
  • 원의 방정식에 대한 기하학 지식
  • Python의 정수 연산 (부동소수점 오차 회피 가능)

Step 2: 문제를 어떤 모델로 바라볼 것인가

“점 (x1,y1)(x_1, y_1)(x1​,y1​)에서 거리 r1r_1r1​인 위치”는 중심이 (x1,y1)(x_1, y_1)(x1​,y1​)이고 반지름이 r1r_1r1​인 원 위의 모든 점입니다.

마찬가지로 점 (x2,y2)(x_2, y_2)(x2​,y2​)에서 거리 r2r_2r2​인 위치는 또 다른 원입니다.

핵심 모델링: 이 문제는 두 원의 교점 개수를 구하는 기하학 문제로 판단.

Step 3: 두 원의 위치 관계 — 케이스 분류

두 원의 관계를 결정하는 핵심 변수는 두 중심 사이의 거리 ddd입니다.

d=(x2−x1)2+(y2−y1)2d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}d=(x2​−x1​)2+(y2​−y1​)2​

엔지니어링 판단: ddd를 직접 구하면 sqrt를 사용해야 하고, 부동소수점 오차가 발생합니다. 대신 d2d^2d2로 비교하면 정수 연산만으로 정확한 판별이 가능합니다.

d2=(x2−x1)2+(y2−y1)2d^2 = (x_2 - x_1)^2 + (y_2 - y_1)^2d2=(x2​−x1​)2+(y2​−y1​)2

이제 케이스를 분류합니다:

Case 1: 두 원이 완전히 같은 경우 → -1 (무한대)

d=0andr1=r2d = 0 \quad \text{and} \quad r_1 = r_2d=0andr1​=r2​

두 원이 완전히 겹치므로 교점이 무한개.

Case 2: 외부에서 만나지 않는 경우 → 0

d>r1+r2d > r_1 + r_2d>r1​+r2​

두 원이 너무 멀리 떨어져서 교점 없음.

Case 3: 내부에서 만나지 않는 경우 → 0

d<∣r1−r2∣d < |r_1 - r_2|d<∣r1​−r2​∣

한 원이 다른 원 내부에 완전히 포함되어 교점 없음.

Case 4: 동심원이지만 반지름이 다른 경우 → 0

d=0andr1≠r2d = 0 \quad \text{and} \quad r_1 \neq r_2d=0andr1​=r2​

중심은 같지만 반지름이 다르면 교점 없음.

Case 5: 외접하는 경우 → 1

d=r1+r2d = r_1 + r_2d=r1​+r2​

Case 6: 내접하는 경우 → 1

d=∣r1−r2∣d = |r_1 - r_2|d=∣r1​−r2​∣

Case 7: 나머지 → 2

∣r1−r2∣<d<r1+r2|r_1 - r_2| < d < r_1 + r_2∣r1​−r2​∣<d<r1​+r2​

두 원이 두 점에서 교차.

Step 4: 부동소수점 회피 전략

여기서 엔지니어링적으로 중요한 판단이 하나 있습니다.

ddd를 구할 때 math.sqrt를 쓰면 부동소수점 오차가 발생합니다. 예를 들어:

import math
# d = 5.0000000000000001 같은 오차 발생 가능
d = math.sqrt((x2 - x1)**2 + (y2 - y1)**2)

하지만 모든 비교를 d2d^2d2 기준으로 변환하면 정수 연산만 사용할 수 있습니다:

원래 조건d2d^2d2 변환
d=r1+r2d = r_1 + r_2d=r1​+r2​d2=(r1+r2)2d^2 = (r_1 + r_2)^2d2=(r1​+r2​)2
d=∥r1−r2∥d = \|r_1 - r_2\|d=∥r1​−r2​∥d2=(r1−r2)2d^2 = (r_1 - r_2)^2d2=(r1​−r2​)2
d>r1+r2d > r_1 + r_2d>r1​+r2​d2>(r1+r2)2d^2 > (r_1 + r_2)^2d2>(r1​+r2​)2
d<∥r1−r2∥d < \|r_1 - r_2\|d<∥r1​−r2​∥d2<(r1−r2)2d^2 < (r_1 - r_2)^2d2<(r1​−r2​)2

모든 입력이 정수이므로 d2d^2d2, (r1+r2)2(r_1 + r_2)^2(r1​+r2​)2, (r1−r2)2(r_1 - r_2)^2(r1​−r2​)2 모두 정수

이것이 이 문제의 핵심 엔지니어링 포인트라고 생각합니다.

Step 5: 구현

import sys
input = sys.stdin.readline

T = int(input())
for _ in range(T):
    x1, y1, r1, x2, y2, r2 = map(int, input().split())

    # 두 중심 사이 거리의 제곱 (정수 연산)
    d_sq = (x2 - x1) ** 2 + (y2 - y1) ** 2

    # 외접/외부 판별용
    sum_r_sq = (r1 + r2) ** 2

    # 내접/내부 판별용
    diff_r_sq = (r1 - r2) ** 2

    if d_sq == 0 and r1 == r2:
        # Case 1: 완전히 같은 원
        print(-1)
    elif d_sq > sum_r_sq:
        # Case 2: 외부에서 만나지 않음
        print(0)
    elif d_sq < diff_r_sq:
        # Case 3: 내부에서 만나지 않음 (동심원 포함)
        print(0)
    elif d_sq == sum_r_sq:
        # Case 5: 외접
        print(1)
    elif d_sq == diff_r_sq:
        # Case 6: 내접
        print(1)
    else:
        # Case 7: 두 점에서 교차
        print(2)

Step 6: 예제 검증

예제 1: 0 0 13 40 0 37

d² = (40-0)² + (0-0)² = 1600
(r1+r2)² = (13+37)² = 2500
(r1-r2)² = (13-37)² = 576

576 < 1600 < 2500 → 두 점에서 교차 → 2 ✓

예제 2: 0 0 3 0 7 4

d² = 0² + 7² = 49
(r1+r2)² = (3+4)² = 49
(r1-r2)² = (3-4)² = 1

d² == (r1+r2)² → 외접 → 1 ✓

예제 3: 1 1 1 1 1 5

d² = 0² + 0² = 0
(r1+r2)² = (1+5)² = 36
(r1-r2)² = (1-5)² = 16

d² == 0이고 r1 ≠ r2 → d² < diff_r_sq (0 < 16) → 내부에서 만나지 않음 → 0 ✓

간단 회고.

1. 모델링이 전부다

“터렛에서 적까지의 거리”라는 스토리를 두 원의 교점으로 모델링을 진행하면 문제는 풀었다고 생각합니다.

실무에서도 마찬가지로 비즈니스 요구사항을 어떤 수학적/알고리즘적 모델로 바라볼수 잇으면 좋을것 같습니다.

2. 부동소수점을 피할 수 있으면 피해라

sqrt를 쓰면 간단해 보이지만, 비교 연산에서 오차가 발생합니다. 정수 입력이 주어지면 d2d^2d2 비교로 변환하는 것이 더 안전하고 빠릅니다. 이건 알고리즘 문제뿐 아니라 실무 수치 계산에서도 동일하게 적용되는 원칙입니다.

그런데 제곱으로 비교하면 정수 크기가 커지는 건 괜찮은가? 이 문제에서는 좌표 범위가 −10,000≤x,y≤10,000-10{,}000 \leq x, y \leq 10{,}000−10,000≤x,y≤10,000이므로:

d² ≤ (20000)² + (20000)² = 800,000,000
(r1+r2)² ≤ (20000)² = 400,000,000

최대 약 8억 — 32비트 정수(약 21억)로도 충분합니다.

하지만 만약 좌표 범위가 10910^9109였다면 d2≤8×1018d^2 \leq 8 \times 10^{18}d2≤8×1018이 되어 64비트 정수의 경계에 걸립니다. C/C++에서는 long long 오버플로우를 주의해야 합니다.

언어정수 오버플로우비고
Python걱정 없음bigint 자동 확장
C/C++int 범위 초과 시 오버플로우long long 사용 필요
Javaint 범위 초과 시long 사용 (최대 9.2×10189.2 \times 10^{18}9.2×1018)

교훈: 부동소수점을 피하기 위해 제곱으로 변환할 때, 값의 범위가 정수 타입 한계를 넘지 않는지 반드시 확인해야 합니다. Python은 이 걱정이 없지만, C/C++에서는 치명적인 버그가 될 수 있습니다.

3. 케이스 분류는 빠짐없이

기하학 문제에서 가장 흔한 실수는 엣지 케이스 누락입니다. 특히:

  • 두 원이 완전히 같은 경우 (무한대)
  • 동심원이지만 반지름이 다른 경우
  • 내접/외접 (정확히 1개 교점)

체계적으로 케이스를 나열하고, 각각에 대해 조건과 결과를 명확히 하는 습관이 중요한것 같습니다.

4. 시간 복잡도

이 풀이의 시간 복잡도는 O(T) — 테스트 케이스당 상수 시간 연산만 합니다. 기하학적 판별이므로 루프나 탐색이 전혀 필요 없습니다.
